## Project Overview
This project focuses on analyzing pizza sales data using MySQL to
understand customer ordering behavior, sales trends, and revenue
distribution. The analysis aims to answer practical business questions
that can help improve decision-making related to product performance
and sales strategy.

All insights are derived using SQL queries involving joins,
aggregations, grouping, and window functions.

---

## Dataset Description
The dataset consists of multiple related tables:

- orders: contains order date and time information
- order_details: contains quantity of pizzas ordered per order
- pizzas: contains pizza size and price details
- pizza_types: contains pizza category and type information

These tables are connected using primary and foreign keys.

## Business Questions Answered
- What is the total quantity ordered for each pizza?
- How are orders distributed across different hours of the day?
- How does pizza demand vary by category?
- What is the average number of pizzas ordered per day?
- Which are the top 3 pizzas based on revenue?
- What percentage of total revenue does each pizza type contribute?
- How does cumulative revenue grow over time?
- What are the top 3 pizzas by revenue within each category?
